# 📦 Blockchain Item Ledger v0.3 (本地防篡改物品登记系统)

这是一个轻量级、自验证、防篡改的本地物品/资产登记系统。它巧妙地将**区块链数据结构（链式哈希）**与**非对称加密签名（Ed25519）**结合在一起，确保账本中的每一条记录、每一张图片都无法被悄无声息地篡改。

在最新的 **v0.3 版本** 中，系统全面打通了物理世界与数字世界的桥梁。通过引入 Telegram Bot 移动端赋能和完整的 QR 码扫码解析闭环，让高深严谨的密码学安全变得触手可及。

## ✨ 核心特性

- **🛡️ 密码学防篡改**：基于 `SHA-256` 算法与 `Ed25519` 私钥签名，确保每一条记录和图片绝对安全，伪造与篡改无所遁形。
- **🔗 幽灵区块与状态流转**：支持严格的链式哈希。独创 `SYSTEM_ACTION` 幽灵区块机制，支持物品状态更新（如标记为售出/消耗），完美遵循区块链“只追加、不修改 (Append-only)”的核心哲学。
- **🤖 Telegram 移动端赋能**：内置 TG Bot 服务端，采用极其严格的 User ID 鉴权拦截。你可以随时随地通过手机发送照片和文字，进行交互式的上链登记或一键检索。
- **📷 物理世界 QR 码闭环**：Web UI 自动为物品生成包含底层 Hash 的专属二维码。支持使用电脑摄像头直接扫码，或向 Telegram 机器人发送二维码图片，系统将瞬间解码并定位物理资产的数字档案。
- **💻 现代数据看板 (Dashboard)**：响应式 Web 界面实时聚合统计“活跃资产总值”与“物品数量”。支持多字段模糊检索（按名称或底层 Hash）、类目过滤、分页加载以及 CSV 账本一键导出。

## 📂 项目结构

item-ledger/
│
├─ database/           # 存储 SQLite 数据库 (ledger.db)
├─ images/             # 存储物品图片 (以图片内容的 SHA-256 命名)
├─ keys/               # 存储 Ed25519 公私钥对 (⚠️ 请妥善保管私钥！)
├─ templates/          # 存储 Flask 前端模板 (index.html)
│
├─ app.py              # Flask Web 服务器入口 & API 路由
├─ tg_bot.py           # Telegram 机器人服务端引擎
├─ ledger.py           # 核心上链逻辑 (计算哈希、读取前区块、私钥签名)
├─ verify.py           # 核心验证逻辑 (哈希比对、链条连续性检查、公钥验签)
│
├─ generate_key.py     # 初始化脚本：生成非对称加密密钥对
├─ init_db.py          # 初始化脚本：创建带区块链结构的数据库表
│
├─ tamper.py           # 攻防测试：模拟黑客绕过应用层强行篡改底层数据库
└─ repair.py           # 攻防测试：上帝视角，使用私钥重写历史修复区块链

🚀 快速开始
1. 环境准备 (Linux/Ubuntu)
本项目依赖于底层的二维码解析 C 语言库。在安装 Python 库之前，请先在系统终端执行：

Bash
sudo apt-get update
sudo apt-get install libzbar0
随后，克隆本仓库并安装 Python 依赖：

Bash
# 建议在 .venv 虚拟环境中执行
pip install -r requirements.txt
2. 配置安全密钥 (.env)
在项目根目录新建一个 .env 文件（此文件已被 .gitignore 忽略，绝不会上传），并填入你的 Telegram Bot 配置：

Plaintext
BOT_TOKEN=你的_Telegram_Bot_Token
ALLOWED_USER_ID=你的_Telegram_用户ID
(提示：如果你不知道 User ID，可以先填 0，启动 tg_bot.py 后向它发送消息，它会拒绝访问并将真实的 ID 打印给你。)

3. 初始化区块链系统
在第一次运行前，需要生成你的专属非对称密钥并初始化数据库创世区块：

Bash
python generate_key.py
python init_db.py
⚠️ 绝对警告：keys/private.key 是你的系统最高权限凭证！只要私钥安全，任何黑客都无法伪造你的账本。请不要泄露给任何人。

4. 启动双端服务
你可以开启两个终端，分别运行 Web UI 和 TG Bot：

Bash
# 启动本地可视化仪表盘 (http://127.0.0.1:5000)
python app.py

# 启动 Telegram 移动端守护进程
python tg_bot.py
💡 使用场景示例
移动入库：购买了新设备，直接拿起手机用 Telegram 向 Bot 发送 /add。按提示依次发送照片、名称、价格。几秒后，Bot 返回绿色的区块 Hash，物品已被永久锚定在你的私人区块链上。

扫码盘点：在 Web UI 点击 "Show QR Code" 打印二维码并贴在设备上。日后只需使用电脑摄像头点击网页端的 "Scan QR"，或用手机拍下二维码发给 Bot，即可瞬间调出该设备的完整购买档案。

资产核销：设备报废或闲置售出。在网页端点击 "Mark Consumed"，系统将在底层追加一条 SYSTEM_ACTION 状态区块。物品在前端变为灰色失效状态，但其历史登记信息与区块链条依然保持连贯完整，随时可供审计。

🔬 原理与攻防测试 (Tamper & Repair)
本系统不仅是一个实用的生产力工具，也是一个极佳的区块链密码学演示沙盒。你可以通过内置的测试脚本，亲自体验数据被破坏与修复的震撼过程。

模拟攻击 (python tamper.py)：脚本将模拟黑客直接篡改 ledger.db 底层数据。此时在网页端点击 "Verify Chain"，系统将立刻爆出红条警告，提示数据哈希不匹配、签名失效、且后续哈希链条全部断裂。

重写历史 (python repair.py)：在拥有私钥的前提下，你可以发动“51% 攻击”。脚本会保留被篡改的数据，以此为基础重新计算全链哈希，并使用私钥重新盖章。网页端验证将再次显示绿色安全通过，展示了“私钥即权力”的区块链真理。

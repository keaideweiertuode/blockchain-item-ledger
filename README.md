# 📦 Blockchain Item Ledger (本地防篡改物品登记系统)

这是一个轻量级、自验证、防篡改的本地物品/资产登记系统。它巧妙地将**区块链数据结构（链式哈希）**与**非对称加密签名（Ed25519）**结合在一起，确保账本中的每一条记录、每一张图片都无法被悄无声息地篡改。配合基于 Flask 和 Bootstrap 5 的现代 Web UI，让枯燥的密码学安全变得易于使用。

## ✨ 核心特性

- **🛡️ 密码学防篡改**：基于 `SHA-256` 算法，为物品信息和图片生成唯一的数字指纹（Hash）。
- **🔗 区块链结构**：每一条记录的哈希都包含上一条记录的哈希（`previous_hash`），牵一发而动全身。
- **✍️ 强数字签名**：使用 `Ed25519` (PyNaCl) 椭圆曲线算法为每个区块使用私钥进行本地签名，防止伪造。
- **🖼️ 跨媒介验证**：不仅验证文本数据，上传的物品图片也会被哈希化并纳入防篡改链条。
- **💻 现代 Web UI**：提供清爽的响应式网页界面，支持类目筛选、模糊搜索和分页浏览。
- **🚨 一键审计**：内置验证脚本，在网页端一键扫描全链，精准定位被篡改的记录和断裂的链条。
- **📊 数据导出**：支持将安全账本一键导出为 CSV/Excel 文件。

## 📂 项目结构

```text
item-ledger/
│
├─ database/           # 存储 SQLite 数据库 (ledger.db)
├─ images/             # 存储物品图片 (以图片内容的 SHA-256 命名)
├─ keys/               # 存储 Ed25519 公私钥对 (请妥善保管私钥！)
├─ templates/          # 存储 Flask 前端模板 (index.html)
│
├─ app.py              # Flask Web 服务器入口 & API 路由
├─ ledger.py           # 核心上链逻辑 (计算哈希、读取前区块、私钥签名)
├─ verify.py           # 核心验证逻辑 (哈希比对、链条连续性检查、公钥验签)
│
├─ generate_key.py     # 初始化脚本：生成非对称加密密钥对
├─ init_db.py          # 初始化脚本：创建带区块链结构的数据库表
│
├─ tamper.py           # 攻防测试脚本：模拟黑客绕过应用层强行篡改底层数据库
└─ repair.py           # 攻防测试脚本：上帝视角，使用私钥重写历史修复区块链

🚀 快速开始
1. 环境准备
确保你的电脑上安装了 Python 3.x。克隆仓库后，安装必需的依赖库：
pip install flask pynacl
2. 初始化系统
在第一次运行前，需要生成你的专属密钥并初始化数据库：

# 生成 Ed25519 公私钥对 (会保存在 keys/ 目录下)
python generate_key.py

# 初始化 SQLite 创世数据库
python init_db.py

⚠️ 警告：keys/private.key 是你的系统最高权限凭证，请绝对保密！只要私钥安全，你的账本就是绝对安全的。

3. 运行 Web 服务
启动基于 Flask 的本地前端服务器：
python app.py
打开浏览器，访问 http://127.0.0.1:5000 即可开始登记你的物品！

🔬 原理与攻防测试 (Tamper & Repair)
本系统不仅是一个工具，也是一个很好的区块链原理教具。 你可以通过内置的测试脚本，亲自体验数据的被破坏与修复。

1. 模拟攻击 (Tamper)
运行以下脚本，系统会模拟黑客直接操作 ledger.db，偷偷篡改最新一件物品的价格数据：

Bash
python tamper.py
结果：回到网页端点击 "Verify Entire Blockchain"，系统会立刻爆出红条警告，提示数据哈希不匹配、签名失效、且后续链条全部断裂。

2. 修复与重写历史 (Repair)
在拥有私钥的前提下，你可以像“51% 攻击”一样重写整个区块链的历史。运行以下脚本：

Bash
python repair.py
结果：系统会保留被黑客修改后的数据，以此为基础重新计算所有哈希，并用你的私钥重新盖章，再次将链条无缝接合。网页端验证将再次显示绿色安全通过。

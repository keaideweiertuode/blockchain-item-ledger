在 Ubuntu 服务器上从 GitHub 拉取项目
一、确认是否安装 Git
先检查系统有没有 Git：git --version
如果显示版本号，例如：
git version 2.43.0
说明已经安装。

如果没有安装：
sudo apt update
sudo apt install git -y

进入你想存放项目的目录，例如
cd /opt
git clone https://github.com/username/project.git

如果作者更新了代码，你只需要：

git pull



如果是私有仓库（需要登录）
有两种方式：

方法1（简单）

输入 GitHub 账号 + Token

方法2（推荐）

使用 SSH Key

例如：git clone git@github.com:username/project.git

建议的服务器项目目录结构
/opt
   ├── project1
   ├── project2
   └── project3

   cd /opt
git clone https://github.com/xxx/xxx.git



Ubuntu 23.04 以后（包括 Ubuntu 24.04）的新机制 PEP 668。
系统 禁止直接用 pip 往系统 Python 里安装包，避免把系统环境搞坏。

方法一（最推荐）使用虚拟环境 venv

先安装完整 Python 组件：

sudo apt install python3-full python3-venv -y
进入你的项目：

cd /usr/blockchain-item-ledger

创建虚拟环境：
python3 -m venv venv

激活环境：
source venv/bin/activate
然后安装依赖：pip install -r requirements.txt

在浏览器里访问项目，有几种方法。
方法一：改绑定地址（推荐）

进入你的项目，找到启动代码，一般是：

app.run()
# 或
app.run(host="127.0.0.1", port=5000)

改成：

app.run(host="0.0.0.0", port=5000)

说明：

0.0.0.0 → 监听所有网卡，外部可以访问

port=5000 → 端口号（可以改）

然后重启项目

可能遇到的第一个问题（很常见）

如果提示：

Permissions 0644 for 'cs15.key' are too open

需要修改私钥权限：

chmod 600 cs15.key

然后再登录：

ssh -i cs15.key ubuntu@1.2.3.4
